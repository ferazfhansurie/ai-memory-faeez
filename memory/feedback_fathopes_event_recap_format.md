The FatHopes event-recap format is **LOCKED**. Use it every time without redesigning.

**Format specs:**
- **9:16 vertical**, FatHopes **lime-green / white / black** brand palette
- **Clean Apple-keynote motion-graphics** feel — warm, celebratory
- **Real event photos stay 100% untouched and do NOT move.** Static photos are background identity; only the typographic infographic (cards, labels, check-marks, the big stat number) animates
- **Hero stat card:** the KG-collected number scales up hard on the strongest beat (e.g. `50KG`, `115KG`). Big numbers read reliably; long text warps
- **Deal wins** shown as animated check-mark rows
- **Copy:** Manglish ~80% Bahasa Melayu (e.g. "minyak terpakai dikumpul hari ni!", "Koperasi FELCRA jadi Hub kutipan baru", "Jumpa lagi, Melaka!")
- **No CTA on recaps** — only a "Terima kasih / Jumpa lagi" sign-off. Exception: Singapore launch video DID include a CTA + contact card because it was a launch, not a recap

**Why:**
The real-event photos are the trust signal — they must stay pixel-identical (real faces, real oil drums, real people). Any animated motion on the photos themselves (Ken Burns, zoom-in) would subtly undermine authenticity.

**How to apply:**
- For new event recaps: confirm KG collected and any deal wins, then apply this template exactly
- If using Seedance to animate: see [[feedback_seedance_cannot_preserve_faces]] — Seedance re-renders and morphs faces. Build recaps in **CapCut or After Effects** (Ken Burns OFF, static photo layer + animated text overlays) or as **animated HTML screen-recorded** for guaranteed pixel-exact photos
- Reserve Seedance for other content types where face fidelity is not required
